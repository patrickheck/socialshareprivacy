  3. Requirements and Limitations

    Technical requirements are jQuery and Javascript enabled in your browser. We have tested with jQuery 1.4.

    If you have suggestions for improvements, please contact us by mail to 2klick@heise.de.

    The permanent activation of the service only works in Internet Explorer version 8, as previous versions do not support JSON. Therefore in IE <= 7 is missing this feature. The rest of the plug-ins is not affected.

    For Facebook an App-ID was needed, but no longer. see Facebook notes.

  4. Dimensions

    The plug-in requires a total of about 600 pixels wide (if all services are running) and about 290 pixels in height, this of course also depends on the length of the specified mouseover text.

    2. Explanation of the code

      <script type="text/javascript" src="jquery.js"></script> 
<script type="text/javascript" src="jquery.socialshareprivacy.js"></script>

      The first line includes the Javascript framework „JQuery“ (http://jquery.com/), the second line of our plug-in. jQuery is not in our package, you have to download it from the website. 

      <script type="text/javascript">
  jQuery(document).ready(function($){
    if($('#socialshareprivacy').length > 0){
      $('#socialshareprivacy').socialSharePrivacy(); 
    }
  });
</script>

      In this <script> the plug-in function is attached to an arbitrary, empty HTML element in the page, in this case, the element with the id socialshareprivacy. In order make sure the item is actually present, we have added the control function if enclosing the attachment to check the conditions.

      <body>
  …
  <div id="socialshareprivacy"></div>
  …
</body>

      Somewhere in <body> section of the website you placed the empty HTML element with the desired id, and used for the same id that you specified in the previous <script>.

  6. Options

    There are several integration options available. In the following, first the general options are listed, and then lists the options for the individual services (Facebook, Twitter, Google+).
    Example of a call with options: 

    $('#socialshareprivacy').socialSharePrivacy({
  services : {
    facebook : {
      'perma_option': 'off'
    }, 
    twitter : {
        'status' : 'off'
    },
    gplus : {
      'display_name' : 'Google Plus'
    }
  },
  'cookie_domain' : 'heise.de'
});
    1. general options
      Option Default Description
      info_link http://www.heise.de/ct/artikel/2-Klicks-fuer-mehr-Datenschutz-1333879.html Link to detailed policy information, in this case, an article on the heise website
      txt_help Text MouseOver-Text of the i icons
      settings_perma Permanently activate the share buttons: Headline of the settings menu
      cookie_path / path where the cookie is valid
      cookie_domain document.location.host domain where the cookie is valid
      cookie_expires 365 how many days the cookie is valid
      css_path socialshareprivacy/socialshareprivacy.css path to the CSS file, if empty no stylesheet will be used.
      uri getURI the URI that is passed on by the Buttons. Can be a constant value (z.B. "http://www.heise.de") or can be a function (see function getURI() in the source code of the plug-in)
    2. Options: Facebook
      Option Default Description
      status on The user will be allowed to select Facebook.
      app_id removed (since Version 1.2) Facebook app ID; is needed to use the Recommend button from Facebook.
      dummy_img socialshareprivacy/images/dummy_facebook.png path to the static images
      txt_info Text MouseOver text for the recommend button
      txt_fb_off not connected with Facebook Text version of the button if Facebook is turned off. Normally not visible for the user.
      txt_fb_on connected with Facebook Text version of the button if Facebook is turned on. Normally not visible for the user.
      perma_option on The user has chosen to permanently turn on Facebook via a cookie.
      display_name Facebook Display name of the service in the options.
      referrer_track   Appended to the end of the URL, can be used to track Referrers.
      language en_US Language Settings
      action recommend Test of the button: Recommend (recommend) or Like (like)
    3. Options: Twitter
      Option Default Description
      status on the User can select Twitter
      dummy_img socialshareprivacy/images/dummy_twitter.png path to the static images
      txt_info Text MouseOver text for the tweet button
      txt_twitter_off not connected with Twitter Text version of the button, if twitter is turned off, normally not visible.
      txt_twitter_on connected with Twitter Text version of the button, if twitter is turned on, normally not visible.
      perma_option on The user has selected to permanently accept twitter via cookie.
      display_name Twitter Display name of the service
      referrer_track   Appended to the end of the URL, can be used to track Referrer.
      tweet_text getTweetText This function checks whether the meta tag DC.title is there and then uses those. If the tag DC.creator is set, then will be appended after a (" - "). Is DC.title does not exist, the <title> tag of the page is used. This option can be overridden with a constant text ( typeof == string ) or a special function ( typeof == function ) that generates the text. The passed text is always truncated to 120 characters and the final spaces are replaced with ….
      language en Language Settings (Default: "en")
    4. Options: Google+
      Option Default Description
      status on The user can select Google+
      dummy_img socialshareprivacy/images/dummy_gplus.png path to the static images

  7. Example Integration

    The following are a few examples of integrations of popular configurations.

    • Only Show Facebook

      $('#socialshareprivacy').socialSharePrivacy({
    services : {
        twitter : {
            'status' : 'off'
        },
        gplus : {
            'status' : 'off'
        }
    }
});

    • Don't offer the option to permanently activate

      $('#socialshareprivacy').socialSharePrivacy({
    services : {
        facebook : {
            'perma_option'  : 'off'
        }, 
        twitter : {
            'perma_option' : 'off'
        },
        gplus : {
            'perma_option' : 'off'
        }
    }
});

    • Only offer Google+ and specify your own path to a CSS file.

      $('#socialshareprivacy').socialSharePrivacy({
    services : {
        facebook : {
            'status' : 'off'
        }, 
        twitter : {
            'status' : 'off'
        }
    },
    'css_path' : '/style/plugins/socialshareprivacy.css'
});

    • Several 2-click button bars on one page

      Option 1: A call to the plug-in with an appropriate selector

      <div class="something">
    <h3><a href="http://www.heise.de">heise</a></h3>
    <p>lorem ipsum</p>
    <div class="social"></div>
</div>

<div class="something">
    <h3><a href="http://www.heise.de/security/">heise security</a></h3>
    <p>dolor sit amet</p>
    <div class="social"></div>
</div>

<script>
$(".social").socialSharePrivacy({
    uri : function(context) {
        return $(context).parents(".something").find("h3 a").attr("href");
    }
});
</script>

      Option 2: Multiple calls to the plug-in

      <div>
    <h3><a href="http://www.heise.de">heise</a></h3>
    <p>lorem ipsum</p>
    <div id="one"></div>
</div>
<script>
$("#one").socialSharePrivacy({
    uri : "http://www.heise.de"
});
</script>

<div>
    <h3><a href="http://www.heise.de/security/">heise security</a></h3>
    <p>dolor sit amet</p>
    <div id="two"></div>
</div>
<script>
$("#two").socialSharePrivacy({
    uri : "http://www.heise.de/security/"
});
</script>

  8. URL

    The URL that is passed to the service can be controlled by an option. By default, a function within the plug-in is used, and the URL of the current page document is taken from document.location.href, however, if the canonical URL is set ( ) then it will be used.

  9. Remember settings

    Before the cookie is requested to check the user preferences, we first check if the plug-in is configured. If the plug-in is changed subsequently, the user has no disadvantages.
    For all services, if the save function is disabled ('perma_option' : 'off') then the settings menu is not displayed.
